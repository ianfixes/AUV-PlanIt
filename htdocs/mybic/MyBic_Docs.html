MyBic User Settings

MyBic provides many settings that will help you take full control of your Ajax applications. Creating AJAX applications that are ready for desktop use is not an easy chore. You have to consider things like the network being down, how to handle JavaScript errors gracefully, etc... MyBic aims to give you complete control over all the possible situations you may run across where the default configuration just doesn't work for you. 

Below is a list of each property you can modify in MyBic and just what it does.

ajaxObj.server_url = server_url;

This setting tells what the path is to the mybic_server.php page on your web server. So if your mybic.js is in the same directory as mybic_server.php then you can just leave this to the default when you create your XMLHTTP object. 

var ajaxObj = new XMLHTTP("mybic_server.php");

ajaxObj.async = true;

By default MyBic sends all AJAX messages asynchronously. This means that when a request is sent to the server the user can still work their way around the web page. Nothing is locked up, the request might even come back 20 minutes later if you're launching a big processing script. If you change this to: ajaxObj.async = false; then requests will be fired off synchronously which means that no further JavaScript processing will take place until that request comes back. Use this for important steps a user must take in your application, or an AJAX call that absolutely must complete before anything is is done. 

ajaxObj.debug = 0;

If you're debugging your ajax application it can sometimes be a daunting task to debug what might be going wrong. Is it the server? Is it your JavaScript code? The first thing I'd recommend is using Firefox and installing "FIREBUG" which is a JS debugging extension. Don't forget to turn on enable XMLHTTPRequests. If you can't use Firefox or you need to debug on a closed Kiosk like system, then set your debug to 1 with the ajaxObj. A little popover will appear on the screen showing your the request/response of your request. Debug has some great features. Read more about it here: http://litfuel.net/mybic/index.html?page=errors
ajaxObj.debug = 1;


ajaxObj.throttle = 1;

You'll probably never want to change this option. This setting means that MyBic will start a queue of your requests. For example if you fire off AJAX request #1. You'll probably want that request to finish before firing off request #2. It basically means you can count on your AJAX requests getting fired in the proper order as they were requested.

ajaxObj.method = "POST";

By default MyBic sends requests as "POST". If you wish to define a request as a "GET" request you just simply need to set the method before the call you wish to make. Since the default is "POST" MyBic will reset to "POST" after each AJAX call. So if you want all your requests to use "GET" you'll need to do the following before every call or only the calls you want to use get:

ajaxObj.method = "GET";
ajaxObj.call("action=helloworld", callBackFunction);

ajaxObj.headers

You can also set custom HTTP headers with MyBic. the ajaxObj.headers property is an array that will contain all the headers you wish to send with your request. By default this is a global setting that will not get reset upon every request. So once you set it, they'll be there for each request. If you want to send a custom HTTP header you can do the following below or visit: http://litfuel.net/mybic/index.html?page=headers for more information.
ajaxObj.headers['jim'] = 'TEST';
ajaxObj.headers['testGUI'] = 'yes';

ajaxObj.format = "JSON";

By default all requests are sent using JSON formatting. This is an ideal setting for receiving arrays of data from PHP but not needed if you're just sending snippets of HTML back. If you want to just use it for text and xml set this to "TEXT". Possible values can be "XML", "TEXT", "JSON". This is reset after every call so you'll need to set this before each call you wish to change from JSON. JSON is now considered to be the preferred method of transmitting data over XML because of ease of use, network packet size and speed. 
ajaxObj.format = 'TEXT';
ajaxObj.format = 'XML';

ajaxObj.abort_timeout = 5000;

When you send an AJAX request MyBic will wait 5 seconds(5000 milliseconds) before calling the network down function. This means that the server has not responded back to the browser within those 5 seconds and we consider the call failed. If you have a server call that you know will take longer you can set this number to something higher accordingly, in milliseconds. This is a per-call basis as well. This will allow you to set some AJAX calls with higher timeouts than others. It will reset back to 5 seconds after each call.

ajaxObj.failed_threshold = 3;

If the abort_timeout is triggered 3 times in a row all Ajax calls will be disabled. This would indicate a major problem with the server or a script that has taken too long 3 times in a row. You can raise or lower this number depending on your needs. If you want 10 failed calls in a row to disable MyBic then just make this setting 10. This is a per-call setting so it will reset to 3 after each call.

ajaxObj.ignoreCall = 0;

If you have debug on and you have a polling Ajax call that launches every second it could clog up your debug pretty quick. If you wish to leave a call of the debugger just set this to 1. It will not show up in the MyBic debugger any longer.

ajaxObj.stopRequest = 0;

There may be occassions that you wish to make an Ajax request then make sure no other Ajax requests are sent until your javascript has completely finished processing. If you set ajaxObj.stopRequest=1; in your call back function, no requests will be sent again until you issue 

ajaxObj.restart();
